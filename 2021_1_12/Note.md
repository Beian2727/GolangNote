##Golang的执行流程分析
****
第一种执行方式： go build
.go文件 ---- go build ----> 可执行文件 ----- 运行 ----> 结果
****
第二种执行方式： go run
.go文件 ---- go run ----> 结果
****
##**区别**
####1、第一种，生成了可执行文件在没有go的开发环境中仍然可以运行
####2、第一种，生成的可执行文件不依赖运行环境，它将库文件打包在了可执行文件之中，所以可执行文件大了很多。
####3、第二种，没有生成可执行文件，而是直接运行了，只有在有go的环境中才可以运行
